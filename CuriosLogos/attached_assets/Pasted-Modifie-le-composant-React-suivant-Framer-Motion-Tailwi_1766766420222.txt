Modifie le composant React suivant (Framer Motion + Tailwind) pour implémenter une section “Cartographie des Médias” avec DEUX scatter plots côte-à-côte et un hover synchronisé entre les deux.

Contraintes générales
- Conserve <motion.section> et le header existant (titre + Badge “Sources”).
- Utilise React + TypeScript si le fichier est TSX.
- Garde Tailwind pour le layout.
- Les deux chartes doivent être dans une grille responsive : 1 colonne sur mobile, 2 colonnes sur desktop (gap correct).
- Ne casse pas l’existant : si les données sont absentes, la section doit rester stable (pas d’erreurs, pas de crash).
- Les médias ont un id unique. Le survol d’un point doit mettre en évidence ce même média sur l’autre charte.

Décision produit à implémenter
- On affiche 2 chartes côte-à-côte :
  1) “Pouvoir & Société”
     X: Libertaire ↔ Autoritaire
     Y: Individualiste ↔ Collectiviste
  2) “Identité & Évolution”
     X: Nationaliste ↔ Mondialiste
     Y: Conservateur ↔ Progressiste
- Les points représentent les médias. Un média a la même couleur sur les deux chartes.
- Interaction hover :
  - Au survol d’un point sur l’une des chartes :
    - point survolé fortement highlight (scale + glow/outline + opacité 100%)
    - point correspondant sur l’autre charte highlight aussi
    - tous les autres points atténués (opacité réduite mais visibles)
  - Tooltip unique (ou tooltip par chart mais même contenu) affichant :
    - nom du média
    - positions sur les 2 chartes (même si on survole une seule charte)
- Interaction click optionnelle mais bienvenue :
  - click sur un point “lock” la sélection (sticky highlight)
  - click dans le vide reset
  - Sur mobile, le tap agit comme hover (et lock si possible)

Implémentation technique demandée
- Remplacer l’unique <TopicScatterPlot> par deux composants (ex: <MediaScatterPlot variant="powerSociety" .../> et <MediaScatterPlot variant="identityEvolution" .../>)
- Ou bien garder TopicScatterPlot mais le rendre paramétrable par variant.
- Centraliser l’état de survol/sélection dans le parent :
  - const [activeMediaId, setActiveMediaId] = useState<string | null>(null)
  - const [lockedMediaId, setLockedMediaId] = useState<string | null>(null)
  - id effectif = lockedMediaId ?? activeMediaId
  - passer onHover/onLeave/onPointClick au plot
- Chaque point doit porter media.id et les charts doivent réagir au même mediaId actif.
- Prévoir un mapping de coordonnées par media :
  - media.positions.powerSociety { x, y }
  - media.positions.identityEvolution { x, y }
  Si ces valeurs n’existent pas, fallback raisonnable :
  - soit 0,0
  - soit n’affiche pas le point + indique “Position inconnue” dans tooltip
  Mais ne crash pas.

UI attendue
- Chaque charte dans une “card” : bg-card, rounded-2xl, border, p-6, shadow-sm.
- Au-dessus de chaque charte : un petit titre (h3) et éventuellement un sous-titre.
- Sous chaque charte : labels des axes X (gauche/droite) en bas, et idéalement labels Y à gauche (optionnel si compliqué).
- Le tout reste cohérent avec le style existant (text-muted-foreground, font-display, etc.).
